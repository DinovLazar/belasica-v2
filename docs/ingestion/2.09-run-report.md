## 5. Publish — owner-directed override of the rights gate (D-2.09R-2)

After the draft ingestion, Lazar directed: *"also publish the new photos not only draft … this is not production we will settle the rights later."* Before acting, the executor surfaced, in full: (a) `belasica-v2.vercel.app` **is** the live public site and reads this exact `production` dataset, so publishing makes the photos public within ~60s; (b) the brief's rights gate is explicit ("every ingested photo stays a draft, full stop"); (c) `facts.md` records Ace's VERIFIED "right to use" but caveats that most photos are third-party screenshots (Facebook / Messenger / newspaper) for which that is legally nuanced, and says to settle it **in writing** before bulk publish; (d) publishing is hard to reverse (crawl/cache/index). Lazar confirmed the informed choice **"ingest as drafts, then publish all."**

- **Executed:** all 881 photo drafts published via a **one-off scratchpad script** (not committed) — the repo's ingestion script (`scripts/ingest/run.mjs`) is untouched and still only ever creates drafts, so the mechanical rights gate remains intact in the codebase.
- **Result:** unauthenticated published photo count is now **889** (8 original demo + 881 ingested). Confirmed via the site's own read: `count(*[_type=="photo"])` → 889.
- **Residual risk (unchanged, on the register):** screenshot-provenance rights are still not settled in writing with Ace. This override does not resolve that; it accepts the exposure. The provisional `provenance` string on every ingested photo still reads НЕПОТВРДЕНИ (unconfirmed).
