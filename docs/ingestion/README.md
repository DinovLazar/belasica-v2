## What the script does (see `scripts/ingest/`)

- `classify.mjs` — pure folder-name → `{ kind, slug, title, decade, flags }`
  classifier implementing `docs/content-ingestion-plan.md` §3 (top-down, first
  match wins). No I/O.
- `run.mjs` — walks the mirror, classifies, generates the report, and (only with
  `--commit`) writes to Sanity.
- `verify-classify.mjs` — asserts `classify()` against the locked mappings; no
  mirror, no Sanity, no token.

Guarantees: dry-run by default (writes nothing); deterministic idempotent IDs
(`season-<slug>`, `drafts.photo-<sha1(relPath)>`) so an interrupted run resumes
without duplicates; **photos created as drafts** (never published — the rights
gate, plan §5); invents nothing beyond slug/title/decade/relatedSeason/image and
the provisional provenance; slug collisions surfaced, never overwritten.

## Prerequisites

1. **Mirror** at `~/belasica-ingest-source/` (or pass `--source <path>`).
2. **Write token** — a Sanity token with **Editor** on `production`, in
   `.env.local` as `SANITY_API_WRITE_TOKEN=` (git-ignored; never committed, never
   `NEXT_PUBLIC`, never on Vercel). Only needed for `--commit`.

## Procedure

```sh
# 0. Verify the classifier (no mirror needed)
node scripts/ingest/verify-classify.mjs

# 1. Dry run — writes the full report, nothing to Sanity. REVIEW BEFORE COMMIT.
node scripts/ingest/run.mjs
#   → writes docs/ingestion/2.09-dry-run-report.md
#   Check: the 5 locked mappings (brief Task 4), 1992-93 = already-present,
#   zero collisions, zero unmatched. Post the report to Lazar before any commit.

# 2. Wave 0 — create the ~74 season shells (slug/title/decade)
node scripts/ingest/run.mjs --commit --wave 0
#   Verify: /arhiva shows the new decades within ~1 min (ISR 60).

# 3. Waves 1–8 — upload photos by decade, as drafts. Run in order.
node scripts/ingest/run.mjs --commit --wave 1   # 1920s–40s
node scripts/ingest/run.mjs --commit --wave 2   # 1950s
# … through --wave 8 (2010s–20s)
#   After each: the script prints the unauthenticated published photo count,
#   which MUST stay 8 (no ingested photo is public).
```

Re-running any completed wave creates zero new documents (deterministic IDs). If
a wave fails partway, just re-run it — it resumes.

Thematic folders (`009. …`, `011. …`, `021. Стадион`, etc.) are classified and
counted but **not uploaded** by default; they need `relatedPerson` links that do
not exist yet. `--include-thematic` stages them with no `relatedSeason` (off by
default this phase).
